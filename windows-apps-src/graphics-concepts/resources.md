# <a name="resources"></a><span data-ttu-id="c48c5-104">Ressourcen</span><span class="sxs-lookup"><span data-stu-id="c48c5-104">Resources</span></span>


<span data-ttu-id="c48c5-105">Eine Ressource ist ein Bereich im Speicher, auf den die Direct3D-Pipeline zugreifen kann.</span><span class="sxs-lookup"><span data-stu-id="c48c5-105">A resource is an area in memory that can be accessed by the Direct3D pipeline.</span></span> <span data-ttu-id="c48c5-106">Damit die Pipeline effizient auf den Speicher zugreifen kann, müssen für die Pipeline bereitgestellte Daten (wie z.B. Input-Geometrie, Shader-Ressourcen und Texturen) in einer Ressource gespeichert werden.</span><span class="sxs-lookup"><span data-stu-id="c48c5-106">In order for the pipeline to access memory efficiently, data that is provided to the pipeline (such as input geometry, shader resources, and textures) must be stored in a resource.</span></span> <span data-ttu-id="c48c5-107">Es gibt zwei Arten von Ressourcen, aus denen alle Direct3D-Ressourcen abgeleitet sind: ein Puffer oder eine Textur.</span><span class="sxs-lookup"><span data-stu-id="c48c5-107">There are two types of resources from which all Direct3D resources derive: a buffer or a texture.</span></span> <span data-ttu-id="c48c5-108">Bis zu 128 Ressourcen können für jede Pipeline-Phase aktiv sein.</span><span class="sxs-lookup"><span data-stu-id="c48c5-108">Up to 128 resources can be active for each pipeline stage.</span></span>

<span data-ttu-id="c48c5-109">Jede Anwendung wird in der Regel viele Ressourcen erstellen.</span><span class="sxs-lookup"><span data-stu-id="c48c5-109">Each application will typically create many resources.</span></span> <span data-ttu-id="c48c5-110">Beispiele für Ressourcen sind unter anderem: Vertexpuffer, Indexpuffer, Konstantenpuffer, Texturen und Shader-Ressourcen.</span><span class="sxs-lookup"><span data-stu-id="c48c5-110">Examples of resource include: vertex buffers, index buffer, constant buffer, textures, and shader resources.</span></span> <span data-ttu-id="c48c5-111">Es gibt mehrere Optionen, die bestimmen, wie Ressourcen verwendet werden können.</span><span class="sxs-lookup"><span data-stu-id="c48c5-111">There are several options that determine how resources can be used.</span></span> <span data-ttu-id="c48c5-112">Sie können Ressourcen erstellen, die stark typisiert sind oder weniger typisieren; sie können steuern, ob Ressourcen sowohl Lese- als auch Schreibzugriff erhalten; sie können Ressourcen nur für die CPU, nur die GPU oder beide zugänglich machen.</span><span class="sxs-lookup"><span data-stu-id="c48c5-112">You can create resources that are strongly typed or type less; you can control whether resources have both read and write access; you can make resources accessible to only the CPU, GPU, or both.</span></span> <span data-ttu-id="c48c5-113">Natürlich wird es einen Geschwindigkeit-Funktionalität-Kompromiss geben - je mehr Funktionen Sie einer Ressource gewähren, desto weniger Leistung sollten Sie erwarten.</span><span class="sxs-lookup"><span data-stu-id="c48c5-113">Naturally, there will be speed vs. functionality tradeoff - the more functionality you allow a resource to have, the less performance you should expect.</span></span>

<span data-ttu-id="c48c5-114">Da eine Anwendung oft viele Texturen verwendet, hat Direct3D ferner das Konzept eines Textur-Arrays, um die Texturverwaltung zu vereinfachen.</span><span class="sxs-lookup"><span data-stu-id="c48c5-114">Since an application often uses many textures, Direct3D also has the concept of a texture array to simplify texture management.</span></span> <span data-ttu-id="c48c5-115">Ein Texturarray enthält eine oder mehrere Texturen (alle desselben Typs und mit denselben Dimensionen), die von einer Anwendung oder von Shadern indiziert werden können.</span><span class="sxs-lookup"><span data-stu-id="c48c5-115">A texture array contains one or more textures (all of the same type and dimensions) that can be indexed from within an application or by shaders.</span></span> <span data-ttu-id="c48c5-116">Durch Texturarrays können Sie eine einzelne Oberfläche mit mehreren Indizes zu verwenden, um auf viele Texturen zuzugreifen.</span><span class="sxs-lookup"><span data-stu-id="c48c5-116">Texture arrays allow you to use a single interface with multiple indexes to access many textures.</span></span> <span data-ttu-id="c48c5-117">Sie können beliebig viele Texturarrays zum Verwalten unterschiedlicher Texturtypen nach Bedarf erstellen.</span><span class="sxs-lookup"><span data-stu-id="c48c5-117">You can create as many texture arrays to manage different texture types as you need.</span></span>

<span data-ttu-id="c48c5-118">Nachdem Sie die Ressourcen erstellt haben, die Ihre Anwendung verwenden wird, verbinden oder binden Sie jede Ressource mit der/an die Pipelinephasen, die diese verwenden.</span><span class="sxs-lookup"><span data-stu-id="c48c5-118">Once you have created the resources that your application will use, you connect or bind each resource to the pipeline stages that will use them.</span></span> <span data-ttu-id="c48c5-119">Dies geschieht durch Aufrufen einer Bindungs-API, die einen Zeiger auf die Ressource verwendet.</span><span class="sxs-lookup"><span data-stu-id="c48c5-119">This is accomplished by calling a bind API, which takes a pointer to the resource.</span></span> <span data-ttu-id="c48c5-120">Da mehr als eine Pipelinephase Zugriff auf dieselbe Ressource benötigen kann, hat Direct3D das Konzept einer Ressourcenansicht.</span><span class="sxs-lookup"><span data-stu-id="c48c5-120">Since more than one pipeline stage may need access to the same resource, Direct3D has the concept of a resource view.</span></span> <span data-ttu-id="c48c5-121">Eine Ansicht identifiziert den Teil einer Ressource, auf den zugegriffen werden kann.</span><span class="sxs-lookup"><span data-stu-id="c48c5-121">A view identifies the portion of a resource that can be accessed.</span></span> <span data-ttu-id="c48c5-122">Sie können *m* Ansichten oder eine Ressource erstellen und diese an *n* Pipelinephasen binden, vorausgesetzt, dass Sie die Bindungsregeln für freigegebene Ressourcen einhalten (die Runtime generiert Fehler zur Kompilierzeit, falls dies nicht der Fall ist).</span><span class="sxs-lookup"><span data-stu-id="c48c5-122">You can create *m* views or a resource and bind them to *n* pipeline stages, assuming you follow binding rules for shared resource (the runtime will generate errors at compile time if you don't).</span></span>

<span data-ttu-id="c48c5-123">Eine Ressourcenansicht bietet ein allgemeines Modell für den Zugriff auf eine Ressource (z.B. Texturen oder Puffer).</span><span class="sxs-lookup"><span data-stu-id="c48c5-123">A resource view provides a general model for access to a resource (such as textures or buffers).</span></span> <span data-ttu-id="c48c5-124">Da Sie der Runtime über eine Ansicht mitteilen können, auf welche Daten diese wie zugreifen kann, können Sie durch Ressourcenansichten weniger Ressourcen typisieren.</span><span class="sxs-lookup"><span data-stu-id="c48c5-124">Because you can use a view to tell the runtime what data to access and how to access it, resource views allow you create type less resources.</span></span> <span data-ttu-id="c48c5-125">Das heißt Sie können Ressourcen für eine bestimmte Größe zur Kompilierzeit erstellen und dann den Datentyp in der Ressource erklären, wenn die Ressource an die Pipeline gebunden wird.</span><span class="sxs-lookup"><span data-stu-id="c48c5-125">That is, you can create resources for a given size at compile time, and then declare the data type within the resource when the resource gets bound to the pipeline.</span></span> <span data-ttu-id="c48c5-126">Ansichten zeichnen sich durch viele Funktionen für die Verwendung von Ressourcen aus, z. B. die Fähigkeit, Tiefe-/Schablonenoberflächen im Shader wiederzugeben, das Generieren von dynamischen Cubemaps in einem Durchgang und das gleichzeitige Berechnen und Ausgeben mehrerer Segmente eines Volumens.</span><span class="sxs-lookup"><span data-stu-id="c48c5-126">Views expose many capabilities for using resources, such as the ability to read back depth/stencil surfaces in the shader, generating dynamic cubemaps in a single pass, and rendering simultaneously to multiple slices of a volume.</span></span>
